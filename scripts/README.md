# scripts/

Reproducible local/CI scripts that the workflows under `.github/workflows/`
call into. Keeping the logic in `scripts/` (instead of inline in YAML)
makes it runnable on a dev box without GHA.

## `parity-inventory.sh`

Implements ADR-0006 §D.1. Pulls the latest prod
`ghcr.io/vfeeg-development/vfeeg-backend` tag, compares its binary
file set / deps / config keys against this fork's source tree,
writes a markdown report.

**Local dry-run:**

```sh
PROD_IMAGE_REF=ghcr.io/vfeeg-development/vfeeg-backend \
FORK_ROOT="$PWD" \
OUT_REPORT=/tmp/report.md \
scripts/parity-inventory.sh
```

Requires `crane`, `redress`, `go`, `python3`, `git` on PATH. To pull
private images, `crane auth login` against ghcr.io first (or have
`~/.docker/config.json` set up).

**CI:** triggered by `.github/workflows/parity-inventory.yml`
(weekly cron + workflow_dispatch). Needs repo secret
`PARITY_PAT_GHCR_READ` — a PAT with `read:packages` scope on the
`vfeeg-development` org. Generated by matthiasm and stored as a
**repo secret** (not org-wide, since this fork is the only consumer).
